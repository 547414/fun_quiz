# 趣味测验平台技术路径分析

> 参考案例：SBTI 人格测试（量子位 2026-04-10）

---

## 一、SBTI 算法拆解

SBTI 的核心算法是**多维向量 + 曼哈顿距离匹配**，分三步：

### Step 1：答题拆维度
- 31 道题，每题 3 选项（A/B/C），得分 1/2/3
- 5 个模型 × 每模型 3 个子维度 = **15 个细分维度**
- 每个维度 2 道题，原始分范围 2～6

| 模型 | 代号 | 子维度 |
|------|------|--------|
| 自我模型 | S | S1 自尊自信 / S2 自我清晰度 / S3 核心价值 |
| 情感模型 | E | E1 依恋安全感 / E2 情感投入度 / E3 边界与依赖 |
| 态度模型 | A | A1 世界观倾向 / A2 规则与灵活度 / A3 人生意义感 |
| 行动驱力 | Ac | Ac1 动机导向 / Ac2 决策风格 / Ac3 执行模式 |
| 社交模型 | So | So1 社交主动性 / So2 人际边界感 / So3 表达与真实度 |

### Step 2：分数归档
- 原始分 2～3 → **L（低）= 1**
- 原始分 4 → **M（中）= 2**
- 原始分 5～6 → **H（高）= 3**
- 每人得到一个 15 维向量，如 `[3,2,3,1,2,...]`

### Step 3：模板匹配
- 预设 **25 种人格模板**（每种也是 15 维向量）
- 逐一计算曼哈顿距离（各维度差值绝对值之和）
- 同时统计「精确命中数」（维度完全一致的数量）
- 排序规则：总差最小 → 精确命中最多 → 相似度最高
- **特殊判定**：隐藏触发题直接跳人格；相似度 < 60% 给兜底人格

## 二、本项目的实现路径

### 2.1 核心数据模型

```
Quiz（测验）
  ├── questions: JSON[]     # 题目列表，含维度映射和选项分值
  ├── dimensions: JSON[]    # 维度定义（15个子维度）
  ├── personas: JSON[]      # 人格模板（25条向量 + 描述文案）
  ├── special_rules: JSON[] # 特殊判定规则（隐藏题、兜底等）
  └── result_config: JSON   # 结果页配置（图片、文案风格）

UserQuizResult（用户答题结果）
  ├── quiz_id
  ├── user_id / session_id  # 支持匿名
  ├── answers: JSON         # 原始答案
  ├── dim_vector: JSON      # 15维归档向量
  ├── persona_code: str     # 命中人格代码
  ├── similarity: float     # 相似度
  └── share_token: str      # 分享唯一码
```

### 2.2 后端 API 设计（FastAPI）

```
POST /api/quiz/{quiz_id}/submit      # 提交答案 → 返回人格结果
GET  /api/quiz/{quiz_id}             # 获取题目（含隐藏维度标记）
GET  /api/quiz/result/{share_token}  # 分享结果页数据

# 管理端
POST /api/admin/quiz                 # 创建测验
PUT  /api/admin/quiz/{id}/personas   # 配置人格模板
GET  /api/admin/quiz/{id}/stats      # 答题统计（各人格分布）
```

### 2.3 前端实现要点

**移动端（mobile/ Vant）**：
- 答题页：进度条 + 单题滑动展示，选项选中后自动跳下一题
- 结果页：人格大图 + 匹配度 + 15维雷达图 + 分享按钮
- 分享机制：生成带 `share_token` 的 H5 链接，支持微信分享

**管理端（web/ Naive UI）**：
- 测验编辑器：题目 / 维度 / 人格模板 CRUD
- 人格模板配置：15维向量可视化输入 + 文案编辑
- 数据看板：人格分布饼图、每日参与量趋势

### 2.4 社交裂变机制（核心传播点）

SBTI 能刷屏的关键是**结果可截图分享**：
- 结果页生成专属图片（人格名 + 匹配度 + 特征标签）
- 后端用 `playwright` 或 `html2canvas` 渲染结果图
- MinIO 存储生成的分享图，URL 短链化

## 三、扩展性设计

本平台定位"各种趣味测验"，测验类型可抽象为：

| 类型 | 算法 | 示例 |
|------|------|------|
| 向量匹配型 | 曼哈顿/余弦距离 | SBTI、MBTI |
| 分支跳题型 | 决策树 | 性格测试、职业测试 |
| 累分型 | 分数段映射结果 | 恋爱脑指数、i人e人程度 |
| 随机型 | 加权随机 | 塔罗、运势 |

在 `Quiz` 的 `questions` JSON 结构中加 `quiz_type` 字段，后端算法层做策略模式路由即可支持多类型。

## 四、技术路径优先级

1. **MVP**：单测验答题 → 结果展示 → 分享图生成（移动端 H5）
2. **管理端**：测验 CRUD + 人格模板编辑 + 基础数据统计
3. **扩展**：多测验类型支持 + 用户历史记录 + 人格组合对比（"我们有多配？"）
4. **增长**：企业微信/微信公众号接入 + 组织内测验（使用 basic_module 的 org/wecom 模块）
